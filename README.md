# 🌐 Web Hands-On

Projeto acadêmico colaborativo para prática de **Git**, **GitHub** e desenvolvimento web com HTML e CSS.

## 📋 Sobre o Projeto

Projeto desenvolvido em equipe com foco em:
- ✅ Controle de versão com Git
- ✅ Colaboração através de branches individuais
- ✅ Pull Requests com revisão obrigatória
- ✅ Desenvolvimento de site em HTML/CSS

## 🎯 Objetivos

- Aplicar boas práticas de versionamento de código
- Entender fluxo de trabalho colaborativo com Git
- Realizar code review entre colegas
- Integrar funcionalidades sem conflitos

## 🚀 Como Começar

### Pré-requisitos
- Git instalado ([download](https://git-scm.com/))
- Editor de código (VS Code, Sublime, etc.)
- Navegador web moderno

### Passos Iniciais

1. **Clone o repositório:**
   ```bash
   git clone <URL-do-repositorio>
   cd web-hands-on
   ```

2. **Crie sua branch pessoal:**
   ```bash
   git checkout -b feature/seu-nome
   ```

3. **Faça suas alterações e commit:**
   ```bash
   git add .
   git commit -m "Descrição clara da mudança"
   ```

4. **Envie para o repositório remoto:**
   ```bash
   git push origin feature/seu-nome
   ```

5. **Abra um Pull Request no GitHub** para revisão

## 🔄 Fluxo de Trabalho

```
main (produção)
  ↓
feature/seu-nome (desenvolvimento)
```

**Processo:**
1. Crie branch a partir de `main`
2. Implementar suas alterações
3. Abrir Pull Request para revisão
4. Após aprovação, faça merge para `main`

## 📝 Convenção de Commits

```
feat: nova funcionalidade
fix: correção de bug
docs: alteração em documentação
style: formatação de código
refactor: refatoração de código
```

**Exemplo:**
```bash
git commit -m "feat: adicionar nova seção na página"
```
